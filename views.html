    Overlay Shim

    This overlay shim will most likely be written in with javascript when a modal is activated. It should be a direct child of body.

    The shim contains overlayShim-content, an element that is fixed just above the bottom of the viewport that can contain "close" or "back to top" prompts.

    We do not block scrolling when a modal is open.

    See .view--modalSnap demo page for example

    <div id="shim" class="overlayShim">
    <div class="overlayShim-content inverted">
        overlayShim content (ie. close prompt)
    </div>
</div>

    Split View

    A "split view" is a pair of view elements, each with a special variant class applied to set the responsiblity of each view as either the "list" component or the "detail" component.

    The list view acts as a scrollable list of selectable items (list of members), while the detail view acts as a viewing area for the selected item (selected member profile).

    A flexParent--row parent is required in order to treat view--splitList and view--splitDetail as flex children.

    Demo

    <div class="flexParent--row">
    <div class="view view--splitList">
        <header class="view-head">
            ... view-head ...
        </header>
        <div class="view-body">
            ... view-body ...
        </div>
    </div>

    <div class="view view--splitDetail">
        <header class="view-head">
            ... view-head ...
        </header>
        <div class="view-body">
            ... view-body ...
        </div>
    </div>
</div>

    Each view in the split view pair requires the view class. The following variant classes assign the view role:

    Class Description
    view--splitList Sets view to "list style" when viewport is wide enough
    view--splitDetail Sets view to "detail style" when viewport is wide enough

    View Base

    A "view" contains all layout and content for a given screen. Typically, a document only has a single view. Split views, where a document contains two view elements are possible (see below).

    A class of .view is required for view elements; the following classes are optional variants:

    Class Description
    view--collection Sets the style for views that are primarily "collections"
    view--loading Displays loading overlay

    Basic view (Demo)

    Basic example of a single view in a document:

    <div class="view">
    <header class="view-head">
        ... view-head ...
    </header>
    <div class="view-body">
        ... view-body ...
    </div>
</div>

    View Body

    The view-body element is our main content container for the view.

    <div class="view">
    <div class="view-body">
        ... stripes, bounds, etc ...
        ... all content ...
    </div>
</div>

    View Head

    The view-head element is the first child (the fixed top nav bar) within every view.

    <div class="view">
    <header class="view-head row">
        <a href="#" class="row-item row-item--shrink valign--middle">[...]</a>
        <a href="#" class="row-item valign--middle">Page Title</a>
        <a href="#" class="row-item row-item--shrink valign--middle">[...]</a>
    </header>
</div>

    view-head variants

    A class of .view-head is required for view headers. The following classes are optional variants (most likely added/removed with javascript to accommodate hero headers).

    Class Description
    .view-head--transparent Makes view header transparent; hides h1 heading
    .view-head--photoOverlay Adds text protext gradient (view-head--trasnparent required with this class)
