    Attachment

    Adding the attachment class to an elemeent creates a context in which layout classes (row-item, chunk, list) use half of their default spacing.

    In foundation, an "attachment" is a piece of content in the feed attached to a parent item (ie. a reply or event meta info).

    Ancestry only affects spacing in attachment. Text sizing remains explicit.

    Discussion reply example

    Member Name

    · 6 mins ago

    Avatar variants

    A base class of avatar is required. The following classes are optional variants:

    Class Description
    avatar--small Adjusts avatar size to the smallest media size
    avatar--big Adjusts avatar size to the biggest media size
    avatar--person Variant for people
    avatar--org Variant for organizers
    avatar--fbFriend Variant for facebook friends
    avatar--icon For icons in avatars (might move this in a later version)
    avatar--noPhoto Resets text-indent for avatars without photos, allowing display of fallback content

    Button variants

    A class of .button is required for buttons. The following classes are optional variants:

    Class Description
    .button--primary rounded button with dark background; for primary actions
    .button--contrast bold, dark text; use on dark or colorful backgrounds or on photos
    .button--small decreases size of button
    .button--fullWidth makes button fill width
    .button--icon button with icon (see example for markup pattern)

    Button base mixin

    Applies base button styles using a mixin.

    (The extend directive can cause ugly cascade problems)

    @include buttonBase();

    Button hover placeholder

    Applies default button :hover, :active, and :focus styles

    // lightens "bgColor" on hover
@mixin buttonHover($bgColor: $C_textPrimary);

    Pill base mixin

    Applies base pill styles using a mixin (for tokens and toggle pills)

    (The extend directive can cause ugly cascade problems)

    @include pillBase();

    Collapsible

    Collapses elements using class toggling. A base class of collapsible is required.

    To collapse the element, add the collapsible--close class.

    NOTE: to hide the collapsible content from screen readers, use display--none when the collapsible is closed.

    Toggle

    I'm collapsible content

    Adding a class of `collapsible--close` to this `section` will set max height to 0vh

    <div class="chunk">
    <a id="cTrigger" class="button button--primary followLink">Toggle</a>
</div>

<section class="stripe collapsible" id="cContainer">
    <div class="bounds collapsible-content">
        <h2>I'm collapsible content</h2>
        <p>Adding a class of `collapsible--close` to this `section` will set max height to 0vh</p>
    </div>
</section>

    Floating Action

    If a page has a single important action, you can use a floatingAction wrapper to call it out. At small screen sizes, the element with the floatingAction class will fix to the bottom of the viewport.

    Note: A class of hasFloatingAction is available if extra space is needed at the bottom of a page for the floating action to "land" in. Apply to the body tag.

    The button below will fix to the bottom of the viewport at small sizes

    Floating Action 
    <div class="bounds">
        <p class="chunk">The button below will fix to the bottom of the viewport at small sizes</p>
        <div class="floatingAction">
            <a href="#" class="button button--primary button--fullWidth">Floating Action</a>
        </div>
    </div>

    Menu

    Context menu style dropdoowns. Actions are delivered in-document next to the menu toggle as a non-js fallback. When a client is javascript-enabled, the menu is hidden and posisitioned fixed (assumes this style of menu will only be found within a view-head).

    Toggling of menu visibility is handled with javascript (supporting js for documentation TK).

    NOTE: This visual example currently shows the non-js fallback only. Use as a guide for markup.

    (icon)
    • Select All
    • Select None 
    <div class="menu">
    <a class="menu-toggle" href="#">(icon)</a>

    <ul role="menu" class="menu-actions">
        <li class="menu-actions-item">
            <a role="menuitem" href="#">Select All</a>
        </li>
        <li class="menu-actions-item">
            <a role="menuitem" href="#">Select None</a>
        </li>
    </ul>
</div>

    Page item variants

    A class of pagination-page is required for pagination list items. The following classes are optional variants:

    Class Description
    .pagination-page--current used to indicate current page
    .pagination-page--control used to indicate the page item as a "previous" or "next" control
